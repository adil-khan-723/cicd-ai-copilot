Where does the build log come from?

The drawer needs full console log for the failed stage. Pick the data source.

A

Fetch live from Jenkins on demand

When you click a build card to open the drawer, the UI calls a new API endpoint GET /api/build-log?job=X&build=N which fetches the console output fresh from Jenkins right then.

✓ Always full log, any length ✓ Works for old builds too ⚠ Requires Jenkins to be reachable when viewing
B

Attach log to the SSE event (stored in memory)

When a build fails, the webhook pipeline already fetches the log. We extend the analysis_complete SSE event to include the full cleaned log (up to 8000 chars). The UI stores it in the build card — no second fetch needed.

✓ Instant — already in memory ✓ Works offline (after event received) ⚠ Capped at 8000 chars (LLM limit) ⚠ Only failed-stage log, not full build
My recommendation: Option A (live fetch). Logs can be very long and the full console is most useful for debugging. The on-demand fetch is one extra API call but gives you the complete picture — not just the truncated excerpt the LLM saw.
