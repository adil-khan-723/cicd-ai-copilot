Tool Crawler data — when does it run?

The crawler checks if Jenkins tools/credentials are properly configured. Currently it runs during failure analysis but results never reach the UI.

A

Attach results to existing analysis_complete event

The crawler already runs during failure processing. Extend the analysis_complete SSE event to include verification data (tool mismatches, missing plugins, missing credentials). No new API call needed — data arrives with the analysis.

✓ Zero extra work — data already computed ✓ No new backend endpoint ⚠ Only runs when a build fails (not on demand)
B

On-demand crawl when drawer opens

When you open the drawer and click the Tool Crawler tab, the UI calls GET /api/verify?job=X&build=N. Backend fetches the Jenkinsfile for that job and runs the crawler fresh against live Jenkins.

✓ Always fresh — reflects current Jenkins state ✓ Works for any job, not just failures ⚠ Needs Jenkinsfile accessible (job config fetch) ⚠ Extra 1-2s wait when tab opens
My recommendation: Option A. The crawler already ran — results are sitting in memory unused. Attaching them to the SSE event is 10 lines of code. Option B is better long-term (works for any job) but adds complexity now. Can always upgrade to B later.
