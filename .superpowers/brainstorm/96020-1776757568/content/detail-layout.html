How should build details open?

When you click a failed build card in the feed — how does the full log + tool crawler appear?

A

Slide-out drawer (right panel)

Feed stays visible on left. Clicking a build slides open a right-side panel with tabs: Logs | Tool Crawler. Close button returns to full feed view.

✓ Feed stays in context ✓ Standard pattern (GitHub PR tabs)
B

Expand inline (accordion)

Clicking the build number expands the card in-place, showing tabs below the summary: Logs | Tool Crawler. No navigation — everything in the feed column.

✓ No context switch ⚠ Gets tall for long logs
C

Full-screen modal/overlay

Clicking opens a full-screen overlay with left sidebar (build info) and main content area split into tabs: Logs | Tool Crawler. Escape/X returns to feed.

✓ Most space for log content ⚠ Loses feed context
